# Smart Exam Proctoring System

An intelligent, real-time online exam proctoring application that monitors candidates through their webcam using state-of-the-art Computer Vision and Deep Learning techniques. Designed with Python, OpenCV, MediaPipe, and YOLOv8, this system detects academic violations, logs them, and automatically captures screenshots of the events.

This project is suitable as a submission for a university-level Computer Vision course.

---

## 📌 Project Overview
Online education demands reliable integrity verification. This application acts as a virtual proctor that operates locally on the user's computer. It runs a lightweight inference pipeline to continuously analyze the video stream for standard cheating indicators, outputting a real-time HUD dashboard and logging violations with photographic evidence.

---

## 🚀 Key Features

1. **Candidate Verification & Presence (Face Detection)**
   - Continuous detection and counting of faces in the webcam feed.
   - Triggers a warning if no candidate is detected for more than **3 seconds**.
   - Triggers an alarm if **multiple persons** are present in the frame.

2. **Head Pose Estimation (Looking Away Detection)**
   - Utilizes a **3D-to-2D Perspective-n-Point (PnP)** projection.
   - Maps 6 primary landmarks from MediaPipe's Face Mesh to a 3D generic facial model.
   - Computes Euler angles (Pitch, Yaw, and Roll).
   - Generates a warning if the candidate looks away (**Left, Right, or Down**) for more than **2 seconds**.

3. **Object Detection (Mobile Phone Detection)**
   - Integrates the **YOLOv8 Nano (neural network)** pre-trained model.
   - Scans frames for cell phones (`cell phone` class in COCO) in real time.
   - Instantly flags any visible mobile phone with a warning banner.

4. **Robust Violation Logging**
   - Automatically maintains a session history file: `violations.csv`.
   - Records: **Timestamp**, **Violation Type**, and **Additional Details** (such as looking direction, face counts, or model confidence).
   - Smart debouncing (cooldowns) prevents duplicate log cluttering during continuous violations.

5. **Evidence Collection (Violation Screenshots)**
   - Saves clean, un-annotated webcam screenshots inside the `screenshots/` directory at the exact moment a violation starts.
   - Filename structure: `violation_YYYYMMDD_HHMMSS.jpg`.

6. **Interactive HUD Dashboard**
   - Semi-transparent, glassmorphic panel overlay displaying current status, face count, look direction, phone status, and total infraction counters.
   - Red visual banners highlight active warnings centered at the top of the video feed.

## 📐 Mathematical Background: Head Pose Estimation
To calculate looking direction without requiring depth cameras, we solve the **Perspective-n-Point (PnP)** problem. 

Given:
- A set of 3D facial feature coordinates (world coordinates, $\mathbf{X}_i$)
- Their corresponding 2D projections in the webcam frame (pixel coordinates, $\mathbf{u}_i$)
- The camera calibration matrix ($K$)

We estimate the rotation vector $\mathbf{r}$ and translation vector $\mathbf{t}$ that minimize the projection error:

$$\sum_{i} \left\| \mathbf{u}_i - \text{project}(\mathbf{X}_i, K, \mathbf{r}, \mathbf{t}) \right\|^2$$

Using OpenCV's `solvePnP` with MediaPipe landmarks:
- **Nose Tip** (Landmark 1) -> mapped to $(0, 0, 0)$
- **Chin** (Landmark 152) -> mapped to $(0, -330, -65)$
- **Right Eye Corner** (Landmark 33) -> mapped to $(-225, 170, -135)$
- **Left Eye Corner** (Landmark 263) -> mapped to $(225, 170, -135)$
- **Right Mouth Corner** (Landmark 61) -> mapped to $(-150, -150, -125)$
- **Left Mouth Corner** (Landmark 291) -> mapped to $(150, -150, -125)$

The rotation vector is transformed into a $3 \times 3$ rotation matrix $R$ using the **Rodrigues Formula**, from which the Euler angles are extracted:
- **Yaw** (rotation around Y-axis): Indicates looking Left (Yaw > 15°) or Right (Yaw < -15°).
- **Pitch** (rotation around X-axis): Indicates looking Down (Pitch < -12°) or Up.

## 💻 Installation & Setup

### Prerequisites
- Python **3.11**
- An active webcam connected to the system.

### Step 1: Clone or copy the project files
Ensure the directories are structured as shown in the directory tree above.

### Step 2: Install dependencies
Open your command terminal (Command Prompt, PowerShell, or bash) inside the `SmartExamProctor/` folder and run:
```bash
pip install -r requirements.txt
```

### Step 3: Run the application
Run the system using Python:
```bash
python main.py
```

*Note: On its first run, the system will automatically download the lightweight `yolov8n.pt` model weights (approx. 6.2 MB) from Ultralytics servers.*
